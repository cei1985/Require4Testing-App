# Prototyp: Require4Testing

Dieses Repository enthält die praktische Ausarbeitung im Rahmen der Fallstudie für den Kurs **„IPWA02-01 – Programmierung von industriellen Informationssystemen mit Java EE“** an der **IU Internationalen Hochschule**.

---

## Projektbeschreibung

Ziel des Projekts ist die Entwicklung eines funktionalen Prototyps für die Web-App **„Require4Testing“**. Die Anwendung dient der Organisation und Verwaltung manueller Anwendertests in Softwareprojekten. 

Der Fokus dieser Arbeit liegt auf der Umsetzung der Kernfunktionalitäten sowie einer persistenten Datenhaltung, um einen realistischen Einblick in den ersten Entwicklungsschritt (Sprint) zu geben.

---

## Ausgewählter Umfang (User Stories)

Gemäß der Aufgabenstellung wurden fünf zentrale Anforderungen aus dem Product Backlog für die prototypische Umsetzung ausgewählt:

1. **Anforderungen erstellen (MUST):** Erfassung von zu testenden Anforderungen durch Requirements Engineers.
2. **Testläufe anlegen (MUST):** Planung und Erstellung von Testzyklen durch Testmanager.
3. **Testfälle erstellen (MUST):** Erstellung spezifischer Testfälle, die einer Anforderung zugeordnet sind.
4. **Zuordnung (SHOULD):** Zuweisung von Testfällen und Testern zu spezifischen Testläufen.
5. **Ergebniserfassung (SHOULD):** Möglichkeit für Tester, die ihnen zugeordneten Testfälle mit einem Ergebnis zu versehen.

---

## Technologiestack

Für die Umsetzung wurde folgender Stack gewählt [3]:

* **Framework:** Spring Boot (Alternative zu JSF/CDI)
* **Persistenz:** JPA / Spring Data JPA mit Hibernate
* **Datenbank:** Relationale Datenbank (MySQL)
* **Frontend:** Pragmatische, funktional ausgerichtete Gestaltung
